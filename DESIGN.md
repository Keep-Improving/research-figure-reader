# Paper Figure Index Design

## Goal

The app should stop treating caption matching as a page-local guess. After a PDF is loaded, the backend should build a paper-level figure index:

- each main figure label
- candidate caption text and page
- body paragraphs that mention or discuss the figure
- source pages and scores
- enough metadata for later AI validation and final figure analysis

The figure index is the shared context for search, evidence review, and image analysis.

## Pipeline

1. Extract all PDF pages into positioned text lines.
2. Detect all main figure caption candidates from `Fig. N`, `Figure N`, and similar starts.
3. Group candidates by figure number.
4. Search the whole paper for non-caption body paragraphs mentioning the same figure.
5. Return a structured index that the frontend can search and inspect.
6. When the user analyzes a page or selected region, use the index entry as context instead of re-guessing from the current page only.

## Data Shape

```ts
type PaperFigureIndex = {
  figures: FigureIndexEntry[]
}

type FigureIndexEntry = {
  figureLabel: string
  captionCandidates: Evidence[]
  bodyEvidence: Evidence[]
  pages: number[]
  score: number
}

type Evidence = {
  pageNumber: number
  text: string
  score?: number
  region?: {
    topY: number
    bottomY: number
    startY: number
  }
}
```

## Matching Policy

Rules should be used for recall and traceability, not as the final authority.

- Caption candidates are generated from full-paper text coordinates.
- Body evidence is generated by full-paper search and excludes caption lines.
- The frontend can search this index immediately.
- A later AI validation step should rank or reject candidates using page images, coordinates, and text.

## Current Implementation Scope

This iteration implements:

- [x] `/api/pdf-index`
- [x] frontend paper index state
- [x] index building after PDF load
- [x] search/filter UI for indexed figures
- [x] selected-figure evidence panel derived from the paper index
- [x] selected caption and top body evidence are sent to the final AI analysis request

This does not yet implement AI validation of the index.

## Review Notes

- The current layer is a retrieval/index layer. It should surface traceable candidates instead of pretending the rule result is always correct.
- The next higher-value step is AI validation/reranking over the indexed candidates and the rendered page image.
- Page-local caption matching remains only a fallback for PDFs where the full-paper index cannot detect a figure.
